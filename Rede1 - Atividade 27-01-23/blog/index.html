1) Qual a função fundamental da camada de transporte?

A camada de transporte trata, fundamentalmente, do processo de conexão entre duas entidades que irão se comunicar.

2) Explique a arquitetura cliente-servidor?

É uma arquitetura de aplicação distribuída, ou seja, na rede existem os fornecedores de recursos ou serviços a rede, que são chamados de servidores, e existem os requerentes dos recursos ou serviços, denominados clientes.

3) Qual mecanismo de endereçamento adotado pela camada de transporte? Como ele funciona? Existe alguma regulamentação?

Utiliza dois protocolos: o TCP e o UDP. O UDP não é orientado à conexão. Ambos os protocolos podem servir a mais de uma aplicação simultaneamente. TCP é um padrão que define como estabelecer e manter uma conversa via rede, em que programas e aplicativos podem trocar dados.

4) Quais as principais características do UDP? Em que situação ele é recomendado?

Ele fornece um serviço com melhor desempenho para um sistema final, a simplicidade do UDP reduz a sobrecarga do uso do protocolo e os serviços podem ser adequados em muitos casos, o UDP serve melhor casos em que a integridade constante dos dados não é essencial.

5) O que é uma conexão? Explique utilizando um esquema ilustrativo.

Uma conexão é uma ligação entre dois pontos, um computador envia dados para o outro usando cabos de rede que conectam os dois computadores. Além disso, os computadores podem se comunicar usando a tecnologia Wi-Fi.

6) Como ocorre o estabelecimento de uma conexão TCP?

Primeiro, uma requisição de conexão é feita pelo cliente para o servidor, em seguida, o servidor responde com uma mensagem de confirmação, a partir daí, os dois sistemas trocam pacotes de dados e informações para estabelecer a conexão, uma vez que a conexão é estabelecida, ambos os sistemas podem enviar e receber dados.

7) Descreva o funcionamento do campo Code Bits.

É uma ferramenta de código que torna mais fácil a criação de conteúdos e programas, ele permite que os codificadores escrevam, modifiquem e criem pequenos pedaços de código, como snippets, snippets de HTML, snippets de JavaScript, snippets de CSS, snippets de PHP, snippets de toda a plataforma .NET e muito mais.

8) Explique de maneira sucinta o controle de sequência adotado pelo TCP.

Utiliza um método de controle de sequência para garantir que as informações sejam enviadas e recebidas em ordem correta, isso é feito atribuindo cada segmento de dados enviado com um número de sequência único.

9) Faça o mesmo com o controle de erros.

O controle de erros é uma técnica de processamento de dados que visa detectar, corrigir, e prevenir erros durante o processamento, transmissão, armazenamento, e apresentação de dados.

10) Como é o funcionamento do protocolo DNS? Descreva utilizando como base sua estrutura hierárquica.

Ele tem uma estrutura hierárquica que consiste em servidores autoritativos, servidores de nomes e servidores de cache, os servidores autoritativos são servidores primários que contêm informações sobre domínios e seus endereços IP, eles geralmente são mantidos por provedores de serviços de Internet.

11) O que é um root server?

É um servidor de nomes que opera na zona de raiz do Sistema de Nome de Domínio (DNS) da Internet, eles respondem diretamente a pedidos para registros armazenados ou armazenados na zona de raiz, eles são fundamentais para a infraestrutura da Internet.

12) Quem são os componentes do DNS e qual a função de cada um deles?

O Domain Name System (DNS) é a estrutura de nomeação responsável por traduzir nomes de domínio em endereços IP, ele é composto por quatro componentes principais: servidores DNS, nomes de domínio, servidores de nomes de domínio e protocolos de resolução de nomes de domínio.

13) Quais os tipos de servidores DNS possíveis?

Member Os servidores DNS podem ser divididos em quatro categorias principais: resolvedores recursivos, servidores raiz, servidores TLD (Top-Level Domain) e servidores autoritativos.

14) Qual a diferença de uma requisição iterativa para uma recursiva?

A diferença entre uma requisição iterativa e uma recursiva é que a requisição iterativa é quando um programa executa um loop que executa a mesma tarefa em sucessão, enquanto a requisição recursiva é quando um programa chama a si mesmo para realizar uma tarefa.

15) Quais as principais características do protocolo FTP?

A capacidade de promover o compartilhamento de arquivos, o encorajamento de servidores remotos como backups, a capacidade de proteger os arquivos dos usuários, possuir um protocolo baseado no TCP, ser um padrão aberto, ter suporte à conexão via linha de comando ou por intermédio de uma interface gráfica.

16) Como o FTP utiliza suas conexões no momento de execução?

O File Transfer Protocol (FTP) utiliza conexões TCP/IP para trocar arquivos. Quando um cliente se conecta a um servidor de FTP, ele se comunica primeiro com o servidor de FTP para estabelecer uma conexão.

17) Faça um breve resumo do protocolo HTTP.

É um protocolo de comunicação da camada de aplicação que fornece meios para a transferência de documentos, como páginas da web e outros arquivos, na internet. É usado principalmente para a transferência de dados entre um servidor web e um navegador da web. Ele permite que o servidor envie páginas HTML, imagens e outros conteúdos ao navegador, e vice-versa.
